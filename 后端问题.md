# 后端需要修复的问题

## 1. CORS 跨域问题（必须修复）

**现象**：前端页面显示"连接服务器超时"，但 `curl http://localhost:8080/health` 返回正常。

**原因**：前端运行在 `localhost:5173`，后端在 `localhost:8080`，浏览器跨域请求被拦截。`curl` 不走浏览器所以不受影响。

**修复方案**：在后端入口文件中加入 CORS 中间件，至少返回以下响应头：

```
Access-Control-Allow-Origin: *
Access-Control-Allow-Headers: Content-Type, Authorization
Access-Control-Allow-Methods: GET, POST, PUT, DELETE, OPTIONS
```

### Express 示例

```javascript
// 方式一：手动设置
app.use((req, res, next) => {
  res.header('Access-Control-Allow-Origin', '*')
  res.header('Access-Control-Allow-Headers', 'Content-Type, Authorization')
  res.header('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE, OPTIONS')
  next()
})

// 方式二：使用 cors 中间件（推荐）
// npm install cors
app.use(require('cors')())
```

### 其他框架

- **Koa**: `npm install @koa/cors`
- **Fastify**: `npm install @fastify/cors`
- **Go (Gin)**: `gin.Default()` 已内置 CORS，或手动 `gin.New().Use(middleware.CORS())`
- **Python (Flask)**: `npm install flask-cors` 或 `from flask_cors import CORS; CORS(app)`

> 注意：OPTIONS 预检请求也要能正确返回 200，否则 PUT/DELETE 请求会被拦截。
